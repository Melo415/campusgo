# CampusGo 工程过程说明

## 项目定位

CampusGo 是面向西电校园生活场景的前后端分离点评应用。项目以商铺浏览、图文笔记和社交互动为核心，支持跨端前端展示与 Spring Boot 后端服务。

本说明根据项目开发文档、当前仓库源码和本地联调截图整理，用于记录可验证的工程过程；未将规划阶段的目标当作已上线能力。

## 测试与缺陷闭环

测试文档记录的执行周期为 2026-05-01 至 2026-05-07，主要采用手动黑盒测试结合接口响应、MySQL 数据核验和回归测试。覆盖范围包括：

- 注册登录、个人资料、笔记发布与展示；
- 点赞、评论、滚动分页和权限控制；
- 前后端数据一致性及移动端/浏览器适配。

文档中记录了重复手机号注册校验、发布后表单清理、消息重复提醒、笔记统计同步等问题，并在回归测试中复核修复结果。README 中的截图保留了其中登录校验、发布—落库、点赞与消息通知的关键证据。

## 配置与安全说明

- 数据库、RabbitMQ 与 OSS 凭证均应通过环境变量配置，禁止提交真实密码或访问密钥；
- 当前账号密码使用带随机盐的 MD5 校验逻辑，适合作为课程项目实现。若后续面向真实用户部署，应迁移到 BCrypt / Argon2，并配合 HTTPS、限流和登录防刷策略；
- 生产环境应收紧跨域来源与 WebSocket 允许来源，并补充自动化接口回归、压力与安全测试。
